# 性能与统计修复说明

本文记录为避免错误计数、降低常驻开销、提升详情页图表渲染效率而做的改动；**未**改动 Markdown `html: true` 等安全相关配置。

**不蒜子整体接入方式（Footer + 详情页）**：见 [`docs/不蒜子接入说明.md`](./不蒜子接入说明.md)。

## 1. 底栏不蒜子：移除定时刷新

**问题**：`FooterSection` 曾每 60 秒调用一次 `requestBusuanziApply()`。不蒜子服务端按 **每次 POST** 累计 PV/UV，定时刷新会导致用户长时间停留在同一页时 **阅读量被重复统计**，并产生无意义的周期性外网请求。

**处理**：去掉 `setInterval` 自动刷新；保留首次进入、**路由变化**、**页签从隐藏回到可见**（`visibilitychange` / `pageshow`）时的刷新逻辑，以及原有的本地缓存与防抖合并（见 `src/utils/busuanziApi.js`）。

**体验说明**：页脚数字不会像「每分钟自动对齐服务端」那样刷新；在用户导航或回到页面时会再次请求并更新展示。

## 2. 图库横向滚动：脱离 Vue 每帧更新

**问题**：原先用 `requestAnimationFrame` 每帧修改两个 `ref`，驱动模板里的 `transform` 绑定，会触发 Vue **每个动画帧**的响应式调度与补丁。

**处理**：

- 位移改为模块内普通变量，通过 `ref` 拿到 `.scroll-track` DOM，在 rAF 内直接设置 `element.style.transform`。
- 当 **两行均处于悬停暂停** 时停止调度下一帧，避免双轨都停时仍空转 rAF。

视觉与交互（悬停暂停、速度、无缝循环）保持不变。

## 3. 项目详情 Mermaid：配置去重与异步代际

**问题**：每次 `htmlContent` 变更都调用 `mermaid.initialize`，重复且无益；快速切换项目或连续替换正文时，多个 `renderMermaidDiagrams` 异步实例可能交错写 DOM。

**处理**：

- 按亮/暗主题缓存「已 initialize 的主题」，仅在主题变化时再次 `initialize`。
- 使用递增 **generation**，在每次 `await` 之后若代际已过期则立刻退出，避免过期任务替换节点。

## 4. 当前系统中值得关注的安全性问题（未在本次修复中改动）

以下条目便于评审与后续加固，**不改变现有业务功能的前提下**仍建议按需处理。

### 4.1 项目详情 Markdown：`html: true` + `v-html`

`ProjectDetail.vue` 使用 `MarkdownIt({ html: true, … })`，并将渲染结果绑定到 `<main class="markdown-body" v-html="htmlContent" />`。

- **风险**：若 Markdown **来源不可信**（例如未来改为用户上传、接口下发），文内原生 HTML 可能构成 **存储型 XSS**。
- **现状**：文档主要来自仓库内静态 `.md`，风险取决于 **谁能改仓库/构建产物**。内部可信源时通常可接受。
- **可选方向**：关闭 `html: true`、或对渲染结果做 HTML 消毒（DOMPurify 等）、或仅允许白名单标签。

### 4.2 Mermaid：`securityLevel: 'loose'` 与 SVG 注入

Mermaid 配置为 `securityLevel: 'loose'`，且流程图等启用 `htmlLabels: true`，生成的 SVG 会通过 `innerHTML` 挂入 Shadow DOM。

- **风险**：在 **恶意图表源码** 场景下，放宽安全级别可能放大脚本或异常 DOM 行为面；通常与 Markdown 信任模型一致。
- **可选方向**：在内容可信前提下维持现状；若来源开放，改为更严格级别并评估 `htmlLabels`。

### 4.3 第三方统计脚本（不蒜子）

向 `cdn.busuanzi.cc` 发送当前 URL 与 referrer；响应键值写入页面时使用 `innerText`，相对安全，但仍属于 **第三方依赖与数据出境/留存策略** 范畴，需在隐私与合规层面自知可控。

### 4.4 其他常规面

- **外链**：页脚等处外链已普遍使用 `rel="noreferrer"`；新增外链时宜保持一致，减轻 `window.opener` 类问题。
- **localStorage**：页脚缓存统计展示用数字，不涉及令牌；仍需避免日后往同一 key 写入敏感信息。
