# 技术分享模块 — 设计文档（对齐站内规范）

> **规范依据**（实现前必读）：  
> - `.cursor/rules/new-module-practical-ux.mdc` — 新模块应对齐 **学习足迹**（`ProjectsSection.vue`）、**作品影像**（`GallerySection.vue`）等标杆。  
> - `.cursor/rules/ui-visual-style.mdc` — 配色 token、`--section-heading-*`、毛玻璃与阴影等。  
> - `.cursor/rules/ui-screen-adaptation.mdc` — `--page-*` 宽度与内边距、断点（如 **900px** 侧栏改纵堆、**768px** 文档类页面分界）、`prefers-reduced-motion`、安全区。  
>
---

## 1. 背景与目标

站点已具备「项目详情」等长文阅读能力；`docs/` 下积累了面向维护者与读者的技术说明。本模块把这些文档以 **「技术分享 / 站内知识库」** 的形式呈现，与作品集叙事互补，**不替代**项目详情页。

**设计原则（保持不变）**

| 原则 | 说明 |
|------|------|
| **内容单一来源** | 正文以 `docs/technical/` 下 Markdown 为准，避免在 Vue 里再写一份长文。 |
| **阅读体验一致** | 版式、代码高亮、暗色模式与 `ProjectDetail` 及全局 token 对齐。 |
| **渐进交付** | MVP：列表 + 单篇；搜索、标签聚合、RSS 等后续迭代。 |

**规范层追加原则**

- **视觉与交互**：不引入与 `main.css` token 割裂的独立主题；区块标题层级复用 **`--section-heading-prefix-size` / `title` / `subtitle`**；卡片与阴影参照 `ProjectsSection` / `GallerySection` 既有写法。
- **实用性**：列表类对齐 **GallerySection**（工具条合一、分页或分块、空状态、边界反馈）；长文 + 侧栏索引对齐 **ProjectsSection**（双栏心智模型、目录跳转与滚动同步高亮、键盘与 `aria-*`）。
- **国际化**：用户可见文案走 **vue-i18n**，键名在 `src/data/cn.js` 与 `src/data/en.js` **同步**维护（与画廊条目注释约定类似）。
- **依赖与风险**：统计若沿用不蒜子，须遵守 [不蒜子接入说明.md](../technical/不蒜子接入说明.md) 的 DOM 与请求约定；不在关键渲染路径依赖无 SLA 外部接口。

## 2. 用户与典型场景

| 角色 | 场景 |
|------|------|
| 访客 | 从导航进入「技术分享」，浏览标题与摘要，点开一篇；必要时用筛选/分页缩小范围。 |
| 作者 | 新增或修改 `docs/technical/` 下文章后构建部署。 |
| 回访者 | 收藏夹或站内链接直达 `/share/:slug`。 |

## 3. 信息架构

### 3.1 路由

| 路径 | 说明 |
|------|------|
| `/share` | 技术分享首页：文章列表（标题、摘要、日期、可选标签）。 |
| `/share/:slug` | 单篇正文，`slug` 与构建期索引中的 id 一致。 |

**导航**：`NavBar` 增加「技术分享」入口，行为与现有锚点/路由一致；文案走 i18n。

### 3.2 内容与 `docs/`

- **纳入范围**：面向读者的技术说明、接入文档、架构/性能笔记等 Markdown。
- **排除**：仅 CI/脚本使用的内部说明 —— 通过 frontmatter `published: false` 或目录排除（技术文档细化）。
- **首发候选**：如 [不蒜子接入说明.md](../technical/不蒜子接入说明.md) 等；是否展示由索引与发布字段控制。

### 3.3 列表页（对齐 GallerySection 心智）

**布局**

- 页顶：**区块标题区**（前缀 + 主标题 + 副标题），样式变量与首页其他 section 一致。
- **工具条（同一视觉簇）**：可选搜索框、标签筛选、排序（默认日期降序）、分页或「加载更多」— 含 `role="toolbar"` 或等价语义，控件带 `aria-label`。
- **主列表**：每条卡片包含 — **标题**、**摘要（1～2 行）**、**日期**（`<time datetime>`）、**标签**、主操作「阅读全文」或整卡点击进入详情。
- **空状态**：无文章 / 筛选无结果时给出文案与「清除筛选」或返回入口（对齐 `emptyFiltered` 类体验）。

**性能**：条目多时 **分页或分块渲染**（定义 `PAGE_SIZE`），避免一次挂载过量 DOM；若路由级较重，参考 `HomeView` **视口哨兵 + 异步组件** 再挂载（见 new-module-practical-ux）。

### 3.4 详情页（对齐 ProjectDetail + ProjectsSection）

**布局**

- **桌面**：**主栏 Markdown 正文** + **可选左侧/右侧目录 rail（TOC）** — TOC 支持点击跳转与滚动同步高亮；宽屏双栏、**900px** 附近改为纵向堆叠（与 `ProjectsSection` / `ProjectDetail` 一致）。
- **移动**：单栏正文；TOC **折叠抽屉**或 **吸底/顶栏内切换**，避免遮挡正文；注意 **`env(safe-area-inset-*)`**。
- **顶栏区域**：标题、日期、标签；**返回列表**（可与现有 `GoBackButton` 模式协调）。
- **正文**：渲染 Markdown；代码高亮、Mermaid 与 `ProjectDetail` 行为一致（含失败降级文案）；阅读宽度使用 **`--page-content-max-width`** 等变量。
- **页脚扩展（二期）**：上一篇 / 下一篇。

### 3.5 主题与 Markdown 皮肤

- 继承站点 **亮/暗**（`html.dark`）与现有 Markdown 样式路径；**禁止**单独 fork 一套与 `github-markdown-css` / 站内正文无关的皮肤。

---

## 4. 国际化（i18n）

- **MVP**：列表页壳层、导航、按钮、空状态、错误提示等 **全部** `t()`；文章正文仍来自 `docs/`（中文为主）。
- **扩展**：英文正文可走独立 md 文件或后续按 slug 映射 locale（另行约定）。
- **键维护**：新增文案时 **同时** 更新 `cn.js` / `en.js`，避免运行时缺键。

---

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| **可维护性** | 新增文章以「在 `docs/technical/` 增加 Markdown」为主；索引由构建期 `import.meta.glob` 生成。 |
| **可发现性** | `/share` 与单篇是否纳入站点地图 / SEO 由你决定；设计上单篇 URL 应可被收录。 |
| **统计** | 不蒜子页级 PV 与列表/详情语义清晰，不与 [不蒜子接入说明.md](../technical/不蒜子接入说明.md) 冲突。 |
| **无障碍** | 主要流程键盘可走通；目录与分页控件具备合理 `aria-*`。 |
| **动效** | 若有过渡动画，须处理 **`prefers-reduced-motion: reduce`**。 |

---

## 6. 里程碑（与规范对齐后的范围）

| 阶段 | 范围 |
|------|------|
| **MVP** | `/share` + `/share/:slug`；构建期索引；frontmatter；列表工具条 + 分页；详情 TOC；全站 NavBar 入口；i18n 壳层；明暗主题与 token。 |
| **二期** | 标签筛选增强、全文搜索、上一篇/下一篇、RSS。 |
| **三期** | 英文镜像、阅读进度、评论或外链 GitHub Discussions。 |

---

## 7. 开放决策（实现前确认）

1. **URL 前缀**：`/share` 或更短如 `/notes`（影响导航文案与外链）。
2. **无 frontmatter 时的默认值**：标题是否取文件名；日期是否取 git 提交时间（自行约定）。
3. **与项目详情边界**：项目页继续承载项目 README；技术分享仅挂 `docs/technical/` 中通用文档，避免重复。
